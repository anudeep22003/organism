1. The Core Shift: From Pipeline to Simulation

Originally, the system looked like this:

Prompt → Story → Characters → Panels → Images

That is a pipeline.

Pipelines are efficient.
Pipelines are brittle.
Pipelines kill emergence.

What you are actually building is this:

World State → Character Agents → Narrative Simulation → Scenes → Media Projections

That is a simulation.

A simulation allows:
	•	Emergence
	•	Dependency
	•	Branching
	•	Multiple outputs
	•	Infinite evolution

This is the first major conceptual shift.

⸻

2. The Vertical Document (The Spatial Anchor)

The stepper UI is procedural.

The vertical document is spatial.

Procedural UI says:

“We are moving forward.”

Spatial UI says:

“This exists all at once.”

Creativity requires spatial awareness.

So the permanent layout is:

Project
  ↓
World (collapsed by default)
  ↓
Characters
  ↓
Scenes
  ↓
Output (Comic, later Audio/Video)

This vertical spine never changes.

No redesign.
No mode switching.
No tab hopping.

Everything expands or collapses within this flow.

Simplicity.

4. Scenes as the Source of Truth

This was a critical insight.

In v1:
Story generates Scenes.

But structurally:

Scenes must become canonical.

Why?

Because:
	•	Scenes are atomic.
	•	Scenes are reorderable.
	•	Scenes are branchable.
	•	Scenes are renderable.
	•	Scenes map cleanly to film language.

Story becomes:

A projection.
A readable summary.
A narrative view.

Not the structural backbone.

That single inversion saves you years.

⸻

5. The Projection Model (Very Important)

This idea resonated with you. Let’s deepen it.

A Scene is pure narrative state:
	•	Characters present
	•	Emotional beats
	•	Actions
	•	Dialogue
	•	Setting

From that same Scene, you can project:

Comic → Still image + speech bubbles
Audio → Voiceover + ambience
Animation → Motion + timing
Film → Scene stitched in timeline

So:

Scene → Projection (Comic)
Scene → Projection (Audio)
Scene → Projection (Video)

Projections are views.
Not truth.

This keeps architecture clean.

It also prevents:

Media logic contaminating narrative logic.

That separation is the foundation of your platform.

⸻

6. Dependency Awareness (Causality Made Visible)

If Ember’s personality changes:
	•	Scenes depending on Ember are affected.
	•	Story projection becomes outdated.
	•	Comic render becomes outdated.

Instead of auto-reflow chaos:

You mark:

⚠ Scene 3 out of sync
⚠ Narrative summary outdated
⚠ Comic render stale

User chooses when to regenerate.

This is Don Norman 101:
Make system causality visible.

Users feel smart when they see consequences.
